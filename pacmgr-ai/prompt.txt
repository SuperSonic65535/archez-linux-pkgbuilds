Your task is to write an update program for Arch Linux. It consists of an update daemon, and a control panel to change settings. The user interface will use GTK4. The program will be written in Python 3 and should support additional languages using language files. It uses polkit for authentication in order to run pacman as root. That same password is also passed to yay.
In this prompt, always treat \n as a line break.

The control panel has the following options:
CheckFrequency: 0-60 seconds/minutes OR 0-24 hours OR 0-7 days OR 0-4 weeks OR 0-12 months (default: 1 week)
CheckDelay: 0-60 seconds/minutes (default: 0 seconds)
ShowNews: true/false (default: true)
AutoDownload: true/false (default: false)
AutoInstall: off/background/shutdown (default: off)
InstallMethod: ask/automatic-background/automatic-terminal/manual-terminal (default: ask)
IncludeAUR: true/false (default: false)
Notifications: true/false (default: true)
Clean cache: true/false (default: false)

The control panel and update daemon reads and writes the configuration in the file $HOME/.config/pacmgr/settings.conf

Notifications will appear either as a pop-up notification in the user's desktop environment, or printed to the TTY if the user is logged into a TTY.
Here are the different notifications:
Updates available: Icon=update-medium-symbolic, message: "Z updates available (X + Y AUR)\npkgname1 pkgname2 pkgname3", button click action: Update, Notification close action: quit update daemon, timeout: none
Downloading pacman packages: Icon=download-symbolic, message: "Downloading X updates\nThis may take a while depending on the number of packages and your internet connection speed.", timeout: 30 seconds
Downloading AUR sources: Icon=download-symbolic, message: "Downloading Y sources\nThis may take a while depending on the total file size and your internet connection speed.", timeout: 30 seconds
Installing pacman packages: Icon=update-busy-symbolic, message: "Installing X updates\nPlease keep your device on. This shouldn't take very long.", timeout: 30 seconds
AUR install: Icon=run-build-symbolic, message: "Building Y AUR updates\Please keep your device on. This may take a while depending on the number of AUR packages and the speed of your device's hardware.", timeout: 30 seconds
Installation complete: Icon=data-success, message: "Update complete\nYour system is now up to date and ready to use.", timeout: 30 seconds
Failure: Icon=data-error, message: "Update failed\nAn error occurred during the update (code: pacman/yay exit code)", timeout: none

If the user is logged into a desktop environment, there should also be a tray indicator icon, which displays status and progress. Here are the different stages for the tray icon:
Waiting/checking for updates: Hidden
Updates available: Icon=update-medium-symbolic, hover message: "Z updates available (X + Y AUR)\npkgname1 pkgname2 pkgname3", left-click action: Update, right-click menu: "Update now; Quit"
Downloading pacman packages: Icon=download-symbolic, hover message: "Downloading X/Y packages\nCurrent: pkgname1 pkgname2 pkgname3\nData: X MiB/Y MiB\nRemaining time: HR:MIN:SEC mins", right-click menu: "Show progress; Pause; Cancel"
Downloading AUR sources: Icon=download-symbolic, hover message: "Downloading X/Y sources\nCurrent: srcfile\nData: X MiB/Y MiB\nRemaining time: HR:MIN:SEC mins", right-click menu: "Show progress; Pause; Cancel"
Installing pacman packages: Icon=update-busy-symbolic, hover message: "Installing X/Y packages\nCurrent: pkgname", right-click menu: "Show progress; Pause; Cancel"
AUR install: Icon=run-build-symbolic, hover message: "Building X/Y AUR packages\nCurrent: pkgbuildname", right-click menu: "Show progress; Pause; Cancel"

Pacman  is used for pacman repository packages, and yay is used for AUR software. The "show progress" option opens a terminal window which shows the output from pacman or yay. If the internet disconnects or times out, the daemon will retry up to 5 times waiting 10 seconds each time before giving up and quitting.

The daemon includes a lightweight terminal window which displays the output of pacman and yay, and allows the user to scroll up and down and enter text. It will have white on black text with colour support. You cannot open new tabs or windows, you cannot use CTRL+C or any other shortcuts, and attempting to close the terminal window will display a message asking "Are you sure?\nIf you cancel the update now, your operating system may become corrupted and unuseable." and if they still click yes, it will stop pacman and yay and the daemon will exit.

The daemon starts as a SystemD user service upon logging in. If the user is not in group wheel, the daemon exits.
Then, if Notifications are set to false AND AutoDownload is false AND AutoInstall is false, the daemon exits.
Next, it reads the timestamp in the file $HOME/.local/share/pacmgr/lastupdate, to how long ago the last update was. If CheckDelay is not set to zero, it will add CheckDelay to the time passed since the last update. If the time passed is less than CheckFrequency, the daemon exits. Otherwise it will check for pacman updates and AUR updates if IncludeAUR is true. If there are no updates, the daemon exits. Otherwise, if AutoDownload is true, show a notification if Notifications are set to true, download all packages from pacman repositories and download AUR sources if IncludeAUR is true.

Afterwards, if Notifications are set to false AND AutoInstall is false, the daemon exits. Otherwise, if Notifications are set to true and AutoInstall is false, show the notification for "Updates available", and wait until the user clicks the "Update" action button. If the user closes the notification instead, the daemon exits.

If ShowNews is true, show a dialog box with the latest news about your installed packages, so the user can read the news in case manual intervention is required. The dialog has two buttons, continue (continue execution) or quit (makes the daemon exit).

When the user clicks Update, or if AutoInstall is true, display the notification for "Installing pacman packages" if Notifications are set to true. Then, depending on the InstallMethod selected:
ask: Ask the user whether they want a silent automatic install (automatic-background), terminal automatic install (automatic-terminal), or terminal install with manual confirmation (manual-terminal)
automatic-background: Silent automatic install in background (first runs "pacman --noconfirm -Syu" and then "yay --noconfirm --answerclean None --answerdiff None --answeredit None --answerupgrade None --removemake --devel --pgpfetch --sudoloop -Syu" if IncludeAUR is true and there are AUR updates available)
automatic-terminal: Automatic install in terminal window (first runs "pacman --noconfirm -Syu" and then "yay --noconfirm --answerclean None --answerdiff None --answeredit None --answerupgrade None --removemake --devel --pgpfetch --sudoloop -Syu" if IncludeAUR is true and there are AUR updates available)
manual-terminal: Install in terminal window with manual confirmation (first runs "pacman -Syu" and then "yay --sudoloop -Syu" if IncludeAUR is true and there are AUR updates available)
If AUR updates are available, display the notification for "AUR install" before running yay, if notifications are true.

While updates are happening, the user may right-click on the daemon's tray icon and pause the update, open a terminal window to view progress, or cancel the update. Additionally, if the file /etc/pacmgr/game exists, there will be an additional option, called "Play a game" - clicking on it will run /etc/pacmgr/game.

If the update has completed successfully, the daemon will write the current time and date to the file $HOME/.local/share/pacmgr/lastupdate.

If Notifications are set to true, play the sound /usr/share/sounds/freedesktop/stereo/complete.oga (if it exists), and show the notification for "Installation complete" if the update has completed successfully, or the notification for "Failure" if an error occurs. Then, the daemon exits.
